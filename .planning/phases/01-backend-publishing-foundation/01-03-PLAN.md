## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-S | Spoofing | `POST /api/v1/approvals/{approval_id}/publish` | mitigate | Require a real backend approval snapshot and reject direct publish requests for unapproved draft versions. |
| T-01-03-T | Tampering | Publish events and attempts | mitigate | Insert append-only events and immutable attempt rows; do not update prior attempt payloads. |
| T-01-03-R | Repudiation | Publish lifecycle audit trail | mitigate | Persist approval ID, draft version ID, attempt number, actor/source, event timestamps, trace ID, and terminal status for each fake job. |
| T-01-03-I | Information Disclosure | Fake provider diagnostics | mitigate | Fake adapter returns redacted diagnostics only and tests fail if secret-like or provider credential fields appear in responses. |
| T-01-03-D | Denial of Service | Publish endpoint | accept | Deterministic local fake jobs are synchronous and bounded; malformed or repeated invalid requests return bounded errors. |
| T-01-03-E | Elevation of Privilege | Fake publisher | mitigate | No OAuth, provider SDK, live provider URL, Postiz endpoint, browser automation, scraping, or cookie-posting code path exists in Phase 1. |
| T-01-SC | Tampering | npm/pip/cargo installs | mitigate | Do not add package-manager installs; verify dependency lists are unchanged. |
</threat_model>
