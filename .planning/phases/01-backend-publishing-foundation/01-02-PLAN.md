## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser to workflow API | `/app` reads backend workflow records and submits approval intent. |
| API response to React state | Backend-owned media refs, channel refs, and approval snapshots become UI state. |
| Browser localStorage boundary | Browser storage remains allowed only for low-risk preferences. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-S | Spoofing | Approval UI | mitigate | UI submits approver data and exact draft version to backend; backend remains source of truth. |
| T-01-02-T | Tampering | Client-side approval snapshot display | mitigate | Display only backend-returned snapshot fields and never rebuild approved payload from mutable client state. |
| T-01-02-R | Repudiation | UI approval history | mitigate | Render backend approver, draft version, approved timestamp, and idempotency suffix. |
| T-01-02-I | Information Disclosure | Browser models and localStorage | mitigate | Browser receives redacted tokenBoundaryRef only and storage helpers omit publish-critical and credential-like records. |
| T-01-02-D | Denial of Service | Initial workflow load | accept | UI shows inline reload state; backend is local bounded skeleton. |
| T-01-02-E | Elevation of Privilege | Browser API client | mitigate | Client cannot create publish jobs or provider calls in this plan; it only loads workflow and requests approval. |
| T-01-SC | Tampering | npm/pip/cargo installs | mitigate | No package-manager installs; dependency lists remain unchanged. |
</threat_model>
