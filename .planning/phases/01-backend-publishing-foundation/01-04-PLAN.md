## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser to publish endpoint | Merchant action queues a fake job from an approved snapshot. |
| API response to timeline UI | Job, attempt, and event records are displayed in the browser. |
| Browser localStorage boundary | Timeline data remains backend-owned and is not persisted as publish-critical browser state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04-S | Spoofing | `Queue fake publish` UI | mitigate | UI submits only backend approval IDs; backend from Plan 03 enforces approval ownership and exact snapshot use. |
| T-01-04-T | Tampering | Timeline display | mitigate | Display normalized backend events and attempts without rebuilding or overwriting attempt history in React. |
| T-01-04-R | Repudiation | Merchant-facing timeline | mitigate | Render backend actor/source, attempt number, timestamp, trace ID when present, and event summary. |
| T-01-04-I | Information Disclosure | Timeline diagnostics | mitigate | Render only backend-redacted diagnostics and never store timeline/job data in localStorage. |
| T-01-04-D | Denial of Service | Workflow refresh | accept | UI disables only the affected queue action and keeps navigation usable while the bounded local API responds. |
| T-01-04-E | Elevation of Privilege | Browser API client | mitigate | Client cannot submit provider targets, credentials, or alternate payloads; fake publisher remains backend-owned. |
| T-01-SC | Tampering | npm/pip/cargo installs | mitigate | No package-manager installs; dependency lists remain unchanged. |
</threat_model>
