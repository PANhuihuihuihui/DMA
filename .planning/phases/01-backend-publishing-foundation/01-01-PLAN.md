<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser to `/api/v1` | Merchant-controlled campaign, draft, and approval input crosses into backend routes. |
| Backend to SQLite | Workflow records, media asset references, token boundary refs, and approval snapshots are persisted outside browser storage. |
| Token boundary to serializers | Server-only token boundary metadata is reduced to redacted public refs before any browser response. |
| Supply chain | Phase 1 backend scaffold uses Python and Node built-ins without package-manager installs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-S | Spoofing | `POST /api/v1/drafts/{draft_id}/approve` | mitigate | Require explicit approver object, exact draftVersionId, and confirmation string; do not infer OAuth/provider identity in Phase 1. |
| T-01-01-T | Tampering | Draft versions, media refs, approval snapshots | mitigate | Store immutable draft versions and media asset refs; approval references one version and serializes a frozen payload that later edits cannot mutate. |
| T-01-01-R | Repudiation | Approval audit trail | mitigate | Persist approver name/email, draft version ID, media refs, approval ID, idempotency key, and timestamps. |
| T-01-01-I | Information Disclosure | Token boundary, API serializers, approval snapshot | mitigate | Public serializers expose redacted tokenBoundaryRef only; tests fail if OAuth secrets, refresh tokens, cookies, authorization headers, API keys, app secrets, or raw provider credentials appear. |
| T-01-01-D | Denial of Service | SQLite workflow endpoints | accept | Phase 1 runs bounded local seeded records; malformed JSON returns bounded 400 responses. |
| T-01-01-E | Elevation of Privilege | Approval route | mitigate | Approval route cannot call provider APIs and live integrations remain absent from this phase. |
| T-01-SC | Tampering | npm/pip/cargo installs | mitigate | Do not add package-manager installs; verify dependency lists remain unchanged. |
</threat_model>
